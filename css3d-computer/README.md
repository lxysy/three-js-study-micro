# css3d-computer — 桌面场景中的坐标变换笔记

## 背景

在 `mesh.js` 中：
- `#desktop`（600×1100 的粉色 div）被转为 `CSS3DObject`，添加在 3D 显示器的屏幕位置
- `.bg` 是一张横图（本质就是 `#desktop` 的子元素），预期视觉效果要占满整个 `#desktop`

但样式里 `.bg` 的 `width: 1100px`、`height: 600px`，正好和 `#desktop` 的 `width: 600px`、`height: 1100px` **相反**，还加了一个 `rotate: -90deg`。

> ⚠️ **注意**：`mesh.js` 中还有 `css3dObj.rotateX(-Math.PI / 2)` 这个 Three.js 的 3D 旋转，和 CSS 的 `rotate(-90deg)` 是两个**不同层面**的旋转，不要混淆。下面先解释 CSS 层面的问题。

---

## 一、核心问题：为什么 `.bg` 的 width/height 要和 `#desktop` 相反？

### 一句话答案

因为 `.bg` 用 `rotate: -90deg` 旋转了。旋转**交换了宽高的视觉表现**，所以 CSS 原始尺寸要和容器相反，旋转后才能对齐。

### 表格对比

| 元素 | CSS width | CSS height | 旋转 | 视觉上的宽 | 视觉上的高 |
|------|-----------|-----------|------|-----------|-----------|
| `#desktop`（容器） | 600px | 1100px | 无 | **600px** | **1100px** |
| `.bg`（子元素） | **1100px** | **600px** | `-90deg` | **600px** | **1100px** |

### 图解

```
容器 #desktop（不旋转）：
┌──────────────────────────────┐
│                              │
│          600px wide          │
│                              │
│                              │ 1100px tall
│                              │
└──────────────────────────────┘

图片 .bg（原始尺寸 1100×600，未旋转时）：
┌──────────────────────────────────────────────────┐
│                                                  │
│                   1100px                         │  600px
│                                                  │
└──────────────────────────────────────────────────┘

图片 .bg（rotate: -90deg 后）：
     ┌──────────────┐
     │              │
     │   600px      │  1100px
     │              │
     └──────────────┘
```

`rotate(-90deg)` 让图片**逆时针旋转 90°**：
- 原始 **1100px 宽** 变成了视觉上的 **高**
- 原始 **600px 高** 变成了视觉上的 **宽**

视觉尺寸就和容器 `#desktop`（600×1100）完全一致了。

### 两个关键点

**① 为什么是 `-90deg`，不是 `90deg`？**

因为 `.bg` 图片是横图（宽 > 高），而容器是竖屏（高 > 宽），需要向左（逆时针）旋转才能匹配：
- `rotate(90deg)` → 顺时针，图片往右倒，**宽高会反**，视觉上是宽 600、高 1100，但图片内容是倒的
- `rotate(-90deg)` → 逆时针，图片往左倒，**宽高对齐**

**② left/top 偏移的计算**

```css
left: -250px;
top: 250px;
```

因为旋转后宽高交换了，**偏移量要按旋转前的原始尺寸算**。容器的中心点在 `(300, 550)`：

```js
left = desktop中心X - bg原始宽度/2 = 300 - 1100/2 = -250
top  = desktop中心Y - bg原始高度/2 = 550 - 600/2  = 250
```

## 二、进阶理解：CSS 坐标轴和 Three.js 坐标轴的关系

这里有一个容易混淆的地方：代码里同时存在**两种旋转**：

| 位置 | 代码 | 旋转类型 | 轴 |
|------|------|---------|----|
| `App.css` | `rotate: -90deg` | CSS 2D 旋转（在屏幕平面内） | 绕元素的 Z 轴（垂直于屏幕） |
| `mesh.js` | `css3dObj.rotateX(-Math.PI / 2)` | Three.js 3D 旋转 | 绕 CSS3DObject 的**局部 X 轴** |

### CSS 2D `rotate(-90deg)` 的坐标系

CSS 2D 变换的坐标系（在屏幕平面上）：

```
           -Y ↑
               │
    -X ←───────┼───────→ +X
               │
           -Y ↓

正角度 (+deg) → 顺时针
负角度 (-deg) → 逆时针
```

所以 `rotate(-90deg)` 是：绕元素中心（Z 轴）逆时针转 90°。

### Three.js `rotateX(-Math.PI / 2)` 的坐标系

CSS3DObject 的局部坐标轴（和 CSS 元素映射关系）：

```
      局部 Y ↑（对应 CSS height）
            │
            │
            ├────────────→ 局部 X（对应 CSS width）
           ↙
        局部 Z（向屏幕外）
```

`rotateX(-Math.PI / 2)` 是：绕元素**宽度方向**（600px 那条边）旋转 -90°，让元素从**竖着**变成**平躺**（XY 平面 → XZ 平面）。

### 为什么需要这两种旋转？

- **`rotate(-90deg)`（CSS）**：适配横图在竖屏容器中的显示，是**纯 UI 层面**的旋转
- **`rotateX(-π/2)`（Three.js）**：把桌面从垂直（贴在显示器屏幕上）变成水平（模拟桌面），是**3D 空间布局**的旋转

两者互不干扰，各管各的。

---

## 三、如果把两种旋转联系起来看

虽然两者是不同层面的旋转，但可以这样理解它们的相似性：

```
CSS rotate(-90deg)：
  width:  1100px → 视觉上变成高
  height: 600px  → 视觉上变成宽

Three.js rotateX(-π/2)：
  CSS height（1100px, 沿 Y） → 翻到 Z 轴方向（深度）
  CSS width（600px, 沿 X）   → 保持不变
```

如果一个元素同时承受这两种旋转（就像你的 `.bg` 作为 `#desktop` 的子元素，而 `#desktop` 又是 CSS3DObject），**视觉效果会叠加**。

---

## 总结

| 问题 | 答案 |
|------|------|
| 为什么 `.bg` 的 w/h 和 `#desktop` 相反？ | 因为 `rotate(-90deg)` 交换了宽高的视觉表现 |
| 方向怎么确定？ | `-90deg` = 逆时针，让横图竖过来对齐容器 |
| left/top 怎么算？ | 按**旋转前**的原始尺寸计算中心偏移 |
| 和 Three.js 的 `rotateX(-π/2)` 有关吗？ | 无关，那是 3D 空间的旋转，让元素从垂直变水平 |
